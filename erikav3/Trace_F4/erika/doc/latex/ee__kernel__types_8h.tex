{\footnotesize\ttfamily \label{ee__kernel__types_8h_a01bbbafc04ff95345be928e1e39c3c68} 
enum \mbox{\hyperlink{ee__kernel__types_8h_a01bbbafc04ff95345be928e1e39c3c68}{Os\+EE\+\_\+trigger\+\_\+status}}}

The trigger statuses have been made to consistently handle race conditions on the update of Alarms and Schedule Tables.

In particular, in the multicore case we have a potential race condition due to the following\+:
\begin{DoxyItemize}
\item the handling of the trigger data structure is done in mutual exclusion using a spin lock.
\item the spinlock is released during the handling of the notifications. This is done because the time spent in handling notifications may be long (e.\+g., a Schedule Table activating a number of tasks at an expiry point.
\item when a core is handling the notifications at an expiry point related to an Alarm (Schedule Table), with the spin lock unlocked, a \doxylink{group__primitives-alarm_ga06987caf9f192223c647e63e1528b8d4}{Cancel\+Alarm()} (\doxylink{group__primitives-schtable_ga71420a854f0ce52d6e4f6aeb28293f78}{Stop\+Schedule\+Table()}) could be issued by another core on the same Alarm (Schedule Table). Once canceled (stopped), the Alarm (Schedule Table) could be restarted again.
\end{DoxyItemize}

Therefore, the status transitions of a trigger will be the following\+:
\begin{DoxyItemize}
\item OSEE\+\_\+\+TRIGGER\+\_\+\+INACTIVE\+: Initial status, trigger inactive
\item OSEE\+\_\+\+TRIGGER\+\_\+\+ACTIVE\+: Alarm or Schedule Table started
\item OSEE\+\_\+\+TRIGGER\+\_\+\+EXPIRED\+: The trigger fired. The status is changed to expired before leaving the critical section.
\item OSEE\+\_\+\+TRIGGER\+\_\+\+CANCELED\+: The status is set to canceled if while the trigger is expired and the spin lock released a \doxylink{group__primitives-alarm_ga06987caf9f192223c647e63e1528b8d4}{Cancel\+Alarm()} or \doxylink{group__primitives-schtable_ga71420a854f0ce52d6e4f6aeb28293f78}{Stop\+Schedule\+Table()} arrives from another core. At the end of the notification handling the status will be set back to OSEE\+\_\+\+TRIGGER\+\_\+\+INACTIVE
\item OSEE\+\_\+\+TRIGGER\+\_\+\+REENABLED\+: The status is set to reenabled if a canceled Alarm or Schedule\+Table is reenabled while a core is still handling the actions. At the end of the notification handling the status will be set to OSEE\+\_\+\+TRIGGER\+\_\+\+ACTIVE with the new start time. 
\end{DoxyItemize}
