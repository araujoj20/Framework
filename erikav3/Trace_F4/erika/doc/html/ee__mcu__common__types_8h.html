◆ OSEE_TYPERET

#define OSEE_TYPERET   OsEE_reg

This type can be used as standard API return type which is shared between the MCU modules.

The OSEE_TYPERET shall normally be used with value E_OK or E_NOT_OK. If those return values are not sufficient, MCU modules specific values can be defined.

◆ StatusType

typedef uint8_t StatusType

Because E_OK is already defined within OSEK, the symbol E_OK has to be shared. To avoid name clashes and redefinition problems, the symbols have to be defined in the following way (approved within implementation):
#ifndef STATUSTYPEDEFINED
#define STATUSTYPEDEFINED
#define E_OK 0x00U
typedef unsigned char StatusType;
#endif
#define E_NOT_OK 0x01U

◆ Std_ReturnType

typedef uint8_t Std_ReturnType

This type can be used as standard API return type which is shared between the RTE and the BSW modules. It shall be defined as follows: typedef uint8 Std_ReturnType; EG: Some DSP architectures have CHAR_BIT equal to 16. So using unsigned char (the smallest unsigned integer usable) is less misleading.
