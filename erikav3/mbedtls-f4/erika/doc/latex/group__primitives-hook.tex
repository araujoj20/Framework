\doxysubsection{Detailed Description}
On multicore systems, hooks are local callbacks to each core.

The \doxylink{group__primitives-hook_ga29f4489d9b6de9008b51f8985046fc3a}{Error\+Hook()} callback is called every time an error is detected inside an ERIKA3 primitive. The callback can be used to implement centralized error handling. A set of macros are also available to better understand the source of the error. In particular, \doxylink{group__primitives-hook_ga29f4489d9b6de9008b51f8985046fc3a}{Error\+Hook()} receives as parameter the error that is raised by the primitive. Then, a call to \doxylink{group__primitives-hook_gaaacd2429e582d3e58f45837b482a1936}{OSError\+Get\+Service\+Id()} returns informations about which primitive caused the error. Finally, calls to the macros OSError\+\_\+\+XXX\+\_\+\+YYY() returns the values of the {\itshape YYY} parameter of the primitive {\itshape XXX}.

The \doxylink{group__primitives-hook_ga18397d2c939dd63649ec1f63213a51b4}{Startup\+Hook()} callback is called inside the \doxylink{group__primitives-startup_ga1c71c32e3d5e1d67037b86488e6410af}{Start\+OS()} primitive to implement the application startup procedure.

The \doxylink{group__primitives-hook_ga9ee2b37172604cbef78d414c503d6e43}{Shutdown\+Hook()} callback is called inside the \doxylink{group__primitives-startup_gaa110c81c8e96f95f3d6780b967a03334}{Shutdown\+OS()} primitive to implement the Application specific shutdown procedures.

The \doxylink{group__primitives-hook_gad0a5498424a4bea39df83c99f947a31b}{Pre\+Task\+Hook()} and \doxylink{group__primitives-hook_ga895e87b2ff6a37e5aeea8814462942f2}{Post\+Task\+Hook()} callbacks are called respectively every time a task becomes the RUNNING Task, and every time a task stops to be the RUNNING Task.

The application can only call a limited set of primitives inside Hook functions.
